# DIRS-FILES-parser-finder
Recursively scans all project files, finds the characters + + + (without spaces), collects this data, and displays it on the client's screen, indicating the line(s) in the specific directory and file where these characters were found. If at least one occurrence of these characters is found in a file, the Windows Explorer launcher icon is displayed

When working on projects, I often come across areas that could use some work. I usually mark these areas with three plus signs (+++). However, this can easily be forgotten later. So, I created a program for myself that scans the selected project directory, finds files containing three plus signs (+++), and then displays them within the existing directory and file structure. Additionally, for convenience, so that I can quickly open the file for editing, icons for launching Windows Explorer are also displayed. Using this button, you can open the file in Explorer, and then open it in another application, such as a text editor.
It's quite possible that similar functionality exists somewhere in Git or PHPStorm. However, I find it more convenient to use this custom program.

Работая над проектами, нередко встречаются места, которые потом следовало бы доработать. Их я обычно отмечаю для себя тремя плюсами: +++. Однако, впоследствии это может забыться. Поэтому для себя я сделал программу, которая сканирует выбранный каталог проекта, находит там файлы, имеющие вхождения трех плюсов (+++), и затем выводит их на экран в составе имеющейся структуры каталогов и файлов. Кроме того, для удобства, чтобы можно было сразу, не тратя время, открыть этот файл для внесения исправлений, показываются также иконки (значки) для запуска Проводника Windows. При помощи этой кнопки можно открыть файл в Проводнике, а там уже - открыть в том или ином приложении, например, в текстовом редакторе.
Вполне возможно, что подобная функциональность присутствует где-нибудь в Git или PHPStorm. Однако, мне удобнее пользоваться этой своей разработкой.
